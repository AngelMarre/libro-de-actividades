Nombres de Ficheros y sus Localizaciones

  • Sufijos de ficheros recomendados y sus localizaciones:
    • Tipo de Fichero Extensión Localización Recomendada
    • JSP .jsp <context root>/<subsystem path>/
    • Fragmento JSP .jsp <context root>/<subsystem path>/
    • Fragmento JSP .jspf <context root>/WEB-INF/jspf/<subsystem path>/
    • Cascading Style Sheet .css <context root>/css/
    • javascript .js <context root>/js/
    • página HTML .html <context root>/<subsystem path>/
    • fuente web .gif, .jpg, etc. <context root>/images/
    • Tag Library Descriptor .tld <context root>/WEB-INF/tld/
  • <context root> es la raíz del contexto de la aplicación web (el directorio raíz dentro de un fichero .war).
  • Se utiliza <subsystem path> para proporcionar agrupación lógica refinada de contenidos estáticos y dinámicos de la página web.
  • Usamos el término fragmento JSP para referirnos a una JSP que se puede incluir en otra JSP. Observa que en JSP 2.0, se utiliza el término "segmento JSP" en lugar de "fragmento JSP". Los fragmento JSP pueden utiliza la extensión .jsp o .jspf, y deberían situarse en /WEB-INF/jspf o con el resto del contenido estático, respectivamente. Los fragmentos JSP que no son páginas completas siempre deberían utilizan la extensión .jspf y situarse en /WEB-INF/jspf.
  • Aunque la especificación JSP recomienda .jspf y .jsp como extensiones posibles para fragmentos JSP, recomendamos usar .jspf mientas .jsf se podría utilizar por la especificación JavaServer Faces_.
  • Situar los ficheros descriptores de librerías de etiquetas y otro contenido no público bajo WEB-INF/ o un subdirectorio bajo éste.
  • Fichero de bienvenida (elemento welcome-file del descriptor web.xml) debería ser index.jsp o index.html.
  • Internacionalización. Por ejemplo, la versión US English /en_US/index.jsp y la versión japonesa /ja_JP/index.jsp.
