<jsp:setProperty>

Pone un valor a la propiedad del bean.

Sintaxis JSP

<jsp:setProperty name="beanInstanceName"	
{ 	
   property="*" |	
   property="propertyName" [ param="parameterName" ] |	
   property="propertyName" value="{stringLiteral| <%= expression %>}"	
}	
/>

Sintaxis XML

<jsp:setProperty name="beanInstanceName"	
{ 	
   property="*" |	
   property="propertyName" [ param="parameterName" ] |	
   property="propertyName" value="{stringLiteral | %= expression %}" 	
}	
/>

Ejemplos

<jsp:setProperty name="mybean" property="*" />	
<jsp:setProperty name="mybean" property="username" />	
<jsp:setProperty name="mybean" property="username" value="Steve" />

Descripción

La etiqueta <jsp:setProperty> da un valor a una o más propiedades del bean.

Atributos

  • name="beanInstanceName"
  • property="*"

Los valores de parámetros enviados al servidor desde los clientes siempre son del tipoString. Los valores del tipo String se convierten a otros tipos y almacenan en las propiedades del bean.

Si el parámetro de la petición tiene valor null, la propiedad del bean no actualiza. Tampoco lo hace si no corresponde el valor con el tipo esperado.

  • property="propertyName" [ param="parameterName" ]

    Si la pripiedad del bean y el parámetro de la petición tienen diferente nombre se deben especificar ambos de la forma property y param. Si tienen el mismo nombre se especifica property y se omite param.

    Si el parámetro es nulo o está vacío la propiedad no se actualiza.

  • property="propertyName" value="{string | <%= expression %>}"

    Poner un valor a una propiedad. El valor puede ser String o una expresión que se evalúa en tiempo de ejecución. If the value is a String.

Recordar

Al usar property="*", las propiedades del bean no son actualizadas en el orden en que aparecen en el formulario HTML o en el bean. SI el orden actualización de las propiedades es importante usar property="propertyName" [ param="parameterName" ]. Mejor: reescribre el bean para que el orden no sea importante.
