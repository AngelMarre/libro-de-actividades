SDL_SemWaitTimeout

Name

SDL_SemWaitTimeout -- Cierra un semáforo, pero espera únicamente durante un tiempo máximo.

Synopsis

#include "SDL.h"
#include "SDL_thread.h"

int SDL_SemWaitTimeout(SDL_sem *sem, Uint32 timeout);

Descripción

SDL_SemWaitTimeout() es una variante de SDL_SemWait pero con un tiempo máximo de espera. Si el valor del semáforo al que apunta sem es positivo (mayor que cero), decrementará atómicamente el valor del semáforo y devolverá 0. En otro caso, esperará hasta timeout milisegundos intentando cerrar el semáforo. Esta función debería ser evitada en la medida de lo posible ya que en ciertas plataformas esta implementada mediante una consulta al semáforo cada milisegundo en un bucle ocupando la CPU.

Tras una llamada con éxito a SDL_SemWaitTimeout() el semáforo puede ser liberado e incrementar atómicamente su contador mediante una llamada con éxito a SDL_SemPost.

Valor Devuelto

Devuelve 0 si el semáforo se cerró con éxito o sinó, bien SDL_MUTEX_TIMEOUT, o bien -1 si el periodo de expiración se excedió o se produjo un error, respectivamente.

Si el semáforo no fu cerrado con éxito, el semáforo no sufrirá ningún cambio.

Ejemplos

res = SDL_SemWaitTimeout(my_sem, WAIT_TIMEOUT_MILLISEC);

if (res == SDL_MUTEX_TIMEOUT) {
        return TRY_AGAIN;
}
if (res == -1) {
        return WAIT_ERROR;
}

...

SDL_SemPost(my_sem);
