SDL_AddTimer

Name

SDL_AddTimer -- Añade un timer que espera un cierto tiempo para depues llamar a la función callback.

Synopsis

#include "SDL.h"

SDL_TimerID SDL_AddTimer(Uint32 interval, SDL_NewTimerCallback callback, void *param);

Callback

/* definición de la "nueva" función callback del timer */
typedef Uint32 (*SDL_NewTimerCallback)(Uint32 interval, void *param);

Descripcion

Añade la función callback que será ejecutada despues del intervalo de tiempo especificado en milisegundos. A la función callback se le pasa el intervalo de tiempo actual y los parametros indicados por el usuario en la función SDL_AddTimer cuando es ejecutada y debe retornar el próximo intervalo de tiempo en la que será de nuevo ejecutada. Si retorna el mismo intervalo con la que fue llamada, el periodo de alarma continua, y si varía, la nueva alarma queda programada.

Para eliminar un timerSDL_RemoveTimer sabiendo el ID (IDentificador) del Timer SDL_AddTimer.

La función callback del timer es ejecutada en un hilo diferente al hilo de el programa principal, y asi no debe llamar a ninguna función contenida en él.

La máxima resolución de este timer es 10 ms, esto significa que si especifica un intervalo de 16 ms en el timer, su función tardará en ejecutarse aproximadamente 20 ms después que halla sido llamada. Si quiere crear un flag que redibuje la pantalla a 30 frames por segundo (siempre 33 ms), deberá seleccionar un timer de intervalo 30ms (mirar el ejemplo). Si usa esta función, necesita pasar SDL_INIT_TIMER a SDL_Init.

Valor Devuelto

Devuelve el ID (IDentificador) del timer añadido o devuelve NULL si ha ocurrido un error durante la creación del timer.

Ejemplos

mi_timer_id = SDL_AddTimer((33/10)*10, mi_callbackfunc, mi_callback_param);
