SDL_AudioCVT

Name

SDL_AudioCVT -- Conversión de la estructura de Audio

Definición de la Estructura

typedef struct{
  int needed;
  Uint16 src_format;
  Uint16 dest_format;
  double rate_incr;
  Uint8 *buf;
  int len;
  int len_cvt;
  int len_mult;
  double len_ratio;
  void (*filters[10])(struct SDL_AudioCVT *cvt, Uint16 format);
  int filter_index;
} SDL_AudioCVT;

Datos de la Estructura

needed (necesitado)Se pone a uno si la conversión es posible
src_format (formato origen)Formato del Audio origen
dest_format (formato destino)Formato del Audio del destino
rate_incr (incremento del ratio)Incremento de la conversión del Ratio
buf (buffer)Buffer de Audio
lenLongitud en bytes del buffer de audio original
len_cvtLongitud en bytes del buffer de audio convertido (calculado)
len_multbuf debe ser len*len_mult bytes en tamaño (calculado)
len_ratioEl tamaño del audio final es len*len_ratio
filters[10](..)Punteros a funciones necesitadas para esta conversión
filter_indexFunción de conversión actual

Descripción

La estructura SDL_AudioCVT se usa para convertir datos de audio entre diferentes formatos. Una estructura SDL_AudioCVT se crea con la función SDL_BuildAudioCVT, mientras que la conversión actual se hace con la función SDL_ConvertAudio.

Muchos de los campos en la estructura SDL_AudioCVT deberían considerarse privados y su función no será discutida aquí.

Uint8 *buf

Esto apunta a los datos de audio que se usaran en la conversión. Es tanto el origen como el destino, lo que significa que los datos de audio convertidos sobre escriben a los datos originales. También significa que los datos convertidos pueden ser mayores (puede haber más) que los datos originales (si se está convirtiendo de 8 bits a 16 bits, por ejemplo), se debe asegurar que buf es suficiente grande. Ver más abajo.

int len

Esto es la longitud en bytes de los datos de audio original.

int len_mult

Como se ha explicado arriba, el buffer de audio necesita ser más grande para almacenar los datos convertidos, los cuales pueden ser más grandes (puede haber más) que los datos de audio original. La longitud de buf debe ser len*len_mult.

double len_ratio

Cuando se ha terminado la conversión de los datos de audio, es necesario saber que cantidad del buffer de audio es válido. len*len_ratio es el tamaño en bytes de los datos de audio. Esto es muy similar a len_mult, sin embargo, cuando los datos de audio convertidos son mas pequeños (hay menos) que los originales len_mult debería ser 1. len_ratio, por otra parte, debería ser un número fraccional entre 0 y 1.
